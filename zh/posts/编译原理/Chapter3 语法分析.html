                                                        # Chapter3 语法分析

                                                        • 正则表达式的能力有限，无法分析具体的语法细节(例如嵌套、anbna^nb^nanbn的n值)，与其等价的有穷自动机同理，因此引入了下推自动机和上下文有关、无关文法

                                                        # 大纲

                                                        • 语法分析的形式化——上下文无关文法
                                                        • 语法分析算法

                                                        # 上下文无关文法(CFG)

                                                        • 是用来描述一个编程语言的语法结构的文法
                                                        • 和正则表达式一样，可以表示递归的规则，而且更强大
                                                        • 是正则表达式的严格超集

                                                        # Chomsky语言层级

                                                        Chomsky hierarchyProduction(产生式)Explanation
                                                        unrestricted(type 0)(自然语言)α→βV+=VN∪VT\alpha \rightarrow \beta \quad V^+=V_N\cup V_Tα→βV+=VN​∪VT​其中α∈V+,  β∈V∗其中\alpha\in V^+,\; \beta\in V^*其中α∈V+,β∈V∗无严格约束
                                                        context sensitive(type 1)(上下文有关文法)αAγ→αβγ,\alpha A \gamma \rightarrow \alpha \beta \gamma,αAγ→αβγ,其中α,  γ∈V∗,  A∈VN,  β∈V+其中\alpha,\;\gamma\in V^*,\;A \in V_N, \; \beta \in V^+其中α,γ∈V∗,A∈VN​,β∈V+在不同的上下文中，AAA可能被不同的γ\gammaγ替换
                                                        context free(type 2)(上下文无关文法)A→β,  A∈VN,A\rightarrow \beta ,\;A\in V_N,A→β,A∈VN​,其中β∈V∗其中\beta \in V^*其中β∈V∗在任何AAA出现的地方都会被γ\gammaγ替换
                                                        regular(type 3)(正则表达式)A→aB  or  A→aA\rightarrow aB\;or\;A\rightarrow aA→aBorA→a(其中A,B  A,b∈VN  ,a∈VT)(其中A,B\;A,b\in V_N\;,a\in V_T)(其中A,BA,b∈VN​,a∈VT​)等价于正则表达式
                                                        • VN和VTV_N和V_TVN​和VT​分别是非终止符号集(nonterminal)和终止符号(terminal)集，是由语言设计者设计的集合，终止符就是常说的token
                                                        • 如果一个符号由它自身定义(a→aa\rightarrow aa→a或int→intint \rightarrow intint→int)那么就是一个终止符号，通常是标点符号，如分号，括号等
                                                        • 如果一个符号有其定义的可再分的结构就是一个非终止符号(letter→[A−Za−z]letter\rightarrow [A-Za-z]letter→[A−Za−z])，通常是句子，短语，表达式等
                                                        • 要注意，A∗A^*A∗(Kleene闭包)是正则表达式特有的规则，CFG中没有，"|"在CFG中表示的是或，用来简化表示多个产生式，而不是正则表达式中的Union。
                                                        • CFG文法规定的第一个文法的左部是开始符号，规定了该语言都满足的一个规则
                                                        • 一个处于较低层级的文法是上级文法的特例，例如RE就是一种特殊的CFG

                                                        # 形式化定义

                                                        CFG:G=(VT,VN,P,S)VT是终止符集合VN是非终止符集合，VN∩VT=∅P是产生式集合，或称语法规则集，满足A→β  A∈VN和β∈(VN∪VT)∗S是初始符号，S∈VN \begin{array}{c} CFG:G=(V_T,V_N,P,S)\\ V_T\text{是终止符集合}\\ V_N\text{是非终止符集合，}V_N\cap V_T=\varnothing\\ P\text{是产生式集合，或称语法规则集，满足}A\rightarrow \beta \;A\in V_N和\beta \in (V_N\cup V_T)^*\\ S\text{是初始符号，}S\in V_N \end{array}CFG:G=(VT​,VN​,P,S)VT​是终止符集合VN​是非终止符集合，VN​∩VT​=∅P是产生式集合，或称语法规则集，满足A→βA∈VN​和β∈(VN​∪VT​)∗S是初始符号，S∈VN​​

                                                        # EBNF(Extended Backus-Naur form)

                                                        • →\rightarrow→：X→Y1Y2Y3...YNX\rightarrow Y_1Y_2Y_3...Y_NX→Y1​Y2​Y3​...YN​表示X可以用Y1Y2Y3...YNX\text{可以用}Y_1Y_2Y_3...Y_NX可以用Y1​Y2​Y3​...YN​来代替，X→εX\rightarrow \varepsilonX→ε表示XXX可以用空串代替，这种A→αA\rightarrow \alphaA→α被称为BNF表示法
                                                        • 简化表示：
                                                          1. 除非特殊说明，否则第一个产生式的左部就是初始符号
                                                          2. 用小写字母表示终止符号
                                                          3. 用大写字符或者<...><...><...>表示非终止符号
                                                          4. 如果左部都为AAA的一系列产生式A→α2,  ...,  A→αn,  A\rightarrow \alpha_2,\;...,\;A\rightarrow \alpha_n,\;A→α2​,...,A→αn​,可以简写为A→α1∣α2∣...∣αnA\rightarrow \alpha_1 | \alpha _2|...|\alpha_nA→α1​∣α2​∣...∣αn​
                                                        • 特别注意：

                                                        S→a∗b(错误写法，没有Kleene闭包)S→AbA→Aa∣ε \begin{array}{c} S\rightarrow a^*b \text{(错误写法，没有Kleene闭包)}\\ S\rightarrow Ab\\ A\rightarrow Aa|\varepsilon \end{array}S→a∗b(错误写法，没有Kleene闭包)S→AbA→Aa∣ε​

                                                        S→a(b∣c)(错误，没有括号)S→aXX→b∣c \begin{array}{c} S\rightarrow a(b|c)(错误，没有括号)\\ S\rightarrow aX\\ X\rightarrow b|c \end{array}S→a(b∣c)(错误，没有括号)S→aXX→b∣c​

                                                        • 左递归：A→Aα∣β  (left  recursive)A \rightarrow A\alpha|\beta\;(left\;recursive)A→Aα∣β(leftrecursive)可表示β,  βα,  βαα...\beta ,\;\beta \alpha,\; \beta \alpha \alpha...β,βα,βαα...即有推导A⇒∗βαn(n=0,1,2,...)A\Rightarrow^*\beta \alpha^n(n=0,1,2,...)A⇒∗βαn(n=0,1,2,...)在EBNF中可表示为A→β{α}A\rightarrow\beta \{\alpha \}A→β{α}
                                                        • 右递归：A→αA∣β  (right  recursive)A \rightarrow \alpha A|\beta\;(right\;recursive)A→αA∣β(rightrecursive)可表示β,  αβ,  αββ...\beta,\; \alpha \beta,\; \alpha \beta \beta...β,αβ,αββ...即有推导A⇒∗αnβ(n=0,1,2,...)A\Rightarrow^* \alpha^n \beta(n=0,1,2,...)A⇒∗αnβ(n=0,1,2,...)在EBNF中可表示为A→{α}βA\rightarrow \{\alpha \}\betaA→{α}β

                                                        # 推导(Derivation)与规约(Reduction)

                                                        • 如果能用一步步推导从初始符号得到需要验证的式子，那么式子就是符合规则的
                                                        • 推导就是不断用产生式的右部来替换一个非终止符
                                                        • ⇒∗\Rightarrow^*⇒∗表示多步推导
                                                        • 由终止符号构成的串称为句子(sentence)，由非终止符号构成的串是句型(sentential form)
                                                        • 以S为开始符号的CFG构成的语言：

                                                          L(G)={s∈VT∗∣  there  exists  S⇒∗s  of  G} L(G)=\{s\in V_T^*|\;there\;exists\;S\Rightarrow^*s\;of\; G\} L(G)={s∈VT∗​∣thereexistsS⇒∗sofG}

                                                        # 语法树

                                                        • 根节点是开始符号
                                                        • 内部节点是非终止符号
                                                        • 叶子节点是终止符号或者ε\varepsilonε
                                                        • 如果节点A有子节点X1,X2,...,XNX_1,X_2,...,X_NX1​,X2​,...,XN​则意为A→X1X2...XNA\rightarrow X_1X_2...X_NA→X1​X2​...XN​
                                                        • 最终的叶子节点连起来就是一个句子
                                                        • 不同的推导会得到不同的树，但是可能会有相同的结果。

                                                        # 最左推导(LeftMost Derivation前缀推导)

                                                        • 总是对句型中最左侧的非终止符号进行一次推导
                                                        • 从开始符号推导到结果，被称为Top-down

                                                        # 最右推导(RightMost Derivation)

                                                        • 从结果反向推回开始符号，这个过程被称为规约s⇒∗Ss\Rightarrow^*Ss⇒∗S
                                                        • 等价于对语法树进行后序遍历的逆过程
                                                        • 最右推导的能力比最左推导要强

                                                        # 抽象语法树

                                                        ![[语法树与抽象语法树.png]]

                                                        • 比起语法树，省略了部分细节，带来了更好的语法抽象，对于后续编译阶段是一个更好的数据结构
                                                        • 它反映了源码token序列的一个抽象，比语法树更高效

                                                        # 歧义(Ambiguity)

                                                        • 对于一个CFG，同样的输入可能有不同的解析 ![[语法树歧义.png]]

                                                        # 解决方法

                                                        • 消除歧义(Disambiguity rule)：不改变文法，列举所有可能造成歧义的情况并进行消除，不现实的
                                                        • 文法重写：改变文法，进行同义转换：(添加优先级，添加关联性)

                                                        E→E−E∣E×E∣(E)∣i  ⇒E→E−E∣TT→T×T∣FF→(E)∣i E\rightarrow E-E|E \times E|(E)|i\;\Rightarrow \begin{array}{c} E\rightarrow E-E|T\\ T\rightarrow T\times T|F\\ F\rightarrow(E)|i \end{array}E→E−E∣E×E∣(E)∣i⇒E→E−E∣TT→T×T∣FF→(E)∣i​

                                                        # 语法分析算法(parsing)

                                                        # Top-Down(Leftmost) parsing

                                                        • 本质上是一个图搜索问题，在树上搜索，查找能否获得一个与输入sentence匹配的路径

                                                        # 回溯算法(Backtracking)

                                                        • 用BFS：进行图遍历搜索，复杂，时间复杂度过高，产生大量无用分支，时间和空间的最差情况都是指数级别。现代编译器中不被使用
                                                        • 剪枝：由终止符号做前缀时，如果无法与输入的前缀匹配则剪枝。(Aa∣Ab∣c→caaaaaAa|Ab|c\rightarrow caaaaaAa∣Ab∣c→caaaaa时，无法剪枝，因为前缀一直是非终止符号)
                                                        • 用DFS：有比BFS更好的空间复杂度和时间复杂度，但是无法匹配(Aa∣c→cAa|c\rightarrow cAa∣c→c，因为会一直循环)

                                                        # 预测推导(predictive parsing)

                                                        • Idea：利用先行词(lokkahead tokens)，也就是上面提到过的终止符前缀
                                                        • 两种分析方法：
                                                        1. 递归下降分析(Recursive-descent parsing)
                                                        2. LL(1)分析

                                                        # 预测分析的概念

                                                        • 从输入串和文法的开始符号开始分析
                                                        • 可以从当前输入的token(s)唯一确定下一个要使用的产生式
                                                        • 预测分析文法包括LL(k)文法，其中L表示从左向右扫描，L表示最左推导，k表示“需要≤k\le k≤k个先行词用于预测 ”
                                                        • LL(1)文法是常用的，也不完全常用

                                                        • 优点：多功能，功能强大，简单有效；灵活，允许程序员安排操作，适用于手工生成的分析器
                                                        • 缺点：必须小心安排每个代码里的操作，而且递归操作会带来大量的时间复杂度和空间复杂度。

                                                        # LL(1)

                                                        • 就是在递归下降的基础上用栈代替了递归，时间复杂度为O(n∣G∣)，n是字符串的长度，G是语法的长度O(n|G|)，n是字符串的长度，G是语法的长度O(n∣G∣)，n是字符串的长度，G是语法的长度 ![[LL(1)原理.png]]
                                                        • 通过分析表(parsing table)来分析怎么转化一个非终止符号(如果不满足LL(1)文法，就会出现一个表项里有多个产生式)
                                                        # 产生分析表
                                                        • 分析表项M[N,t]M[N,t]M[N,t]表示对于一个非终止符号N，当前输入token为t时，选择的产生式
                                                        • 构造分析表，对每个产生式A→αA\rightarrow \alphaA→α重复以下两步：
                                                          1. 对每个在FIRST[α]对每个在FIRST[\alpha]对每个在FIRST[α]中的tokentokentoken，把产生式A→αA\rightarrow\alphaA→α加入表项M[A,a]中M[A,a]中M[A,a]中
                                                          2. 如果ε∈First(α)\varepsilon \in First(\alpha)ε∈First(α)，对Follow(A)中的每个元素(包括token和$)Follow(A)\text{中的每个元素(包括token和}\$)Follow(A)中的每个元素(包括token和$)，把A→αA\rightarrow \alphaA→α加入表项M[A,a]M[A,a]M[A,a] ![[构建LL(1) parsing table 1.png]] ![[构建LL(1) parsing table 2.png]]
                                                          • 从分析表的角度来说，一个不满足LL(1)文法的CFG文法产生的产生表的一个表项内可能有多个产生式，无法做到唯一选择
                                                        # 分析步骤
                                                        1. 开始于把起始符号压入栈中；
                                                        2. 把栈顶的非终止符号A用查表得到的产生式A→αA\rightarrow \alphaA→α替换；
                                                        3. 当栈顶元素为终止符号后，将栈顶元素与输入匹配，匹配成功后，同时丢弃栈顶元素和输入元素
                                                        4. 重复2、3步，直到输入和栈同时为空，则分析成功结束。
